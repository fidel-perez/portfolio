# Microsites — per-target proposal pages

Each subfolder is one tailored microsite for one outreach target.
GitHub Pages serves the whole `portfolio` repo from the root, so any
`microsites/<slug>/index.html` is automatically live at:

```
https://fidel-perez.github.io/portfolio/microsites/<slug>/
```

No Pages config change needed — drop a folder, push, the URL works.

## Slug convention

| Origin                   | Slug shape                          | Example                              |
|--------------------------|-------------------------------------|--------------------------------------|
| `fidel-jobs` (job apps)  | `<company-slug>`                    | `microsites/acme-robotics/`          |
| `felix-vino` bodega      | `vendvino-<bodega-slug>`            | `microsites/vendvino-akilia/`        |
| `felix-vino` restaurant  | `vendvino-<restaurant-slug>`        | `microsites/vendvino-saddle/`        |

Rules:
- kebab-case, ASCII only.
- Strip company-suffix noise (`-sl`, `-bodegas`, etc.) for cleaner URLs.
- The `vendvino-` prefix on the wine campaigns avoids collisions with
  hiring targets that share a name.

## How they're produced

These pages are NOT hand-written. They are auto-generated by a private
agent platform from declarative inputs:

1. **Strategy** — `paperclip-customization/companies/<company>.yaml` →
   `microsite_strategies.<mode>` defines block order + theme + static args.
2. **Block library** — `paperclip-customization/microsite-blocks/`
   provides the canonical HTML partials (hero, competitor signals,
   wine selection, CTA, footer).
3. **Per-target args** — the research crew populates the
   `filled_by: research_crew` slots; the dev crew calls
   `render_fixture.py` (or its production equivalent) and opens a PR
   on this repo with the rendered `microsites/<slug>/index.html`.

So this directory is the *output*; the input lives in the agent
platform repo (private). The output being public + reproducible is the
demonstration.

## Adding by hand

If you ever need to drop one in manually (e.g. for testing the URL):

```bash
mkdir microsites/<slug>
cp microsites/_template.html microsites/<slug>/index.html  # if a template exists
# or render via:
#   python3 ../path/to/microsite-blocks/_fixtures/render_fixture.py \
#     ../path/to/<fixture>.yaml microsites/<slug>/index.html
```

Push, wait ~60s for Pages, the URL is live.
